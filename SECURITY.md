# 🔒 Política de Segurança

Obrigado por ajudar a manter o Getfy seguro.

## 🛡️ Como reportar vulnerabilidades

Se encontrar uma falha de segurança, não abra uma issue pública com detalhes exploráveis.

Prefira reportar diretamente aos mantenedores do projeto pelo GitHub:

- Perfil oficial: https://github.com/getfy-opensource
- Autor/Desenvolvedor: https://github.com/LeonardoIsrael0516
- Contribuidor/Desenvolvedor: https://github.com/alexbritodev

Inclua, quando possível:

- Descrição clara do problema.
- Passos para reprodução.
- Impacto esperado.
- Versão, branch ou commit afetado.
- Evidências seguras, sem expor dados sensíveis.

## ✅ Boas práticas esperadas

- Não exponha tokens, senhas, chaves Pix, credenciais de gateway ou dados de clientes.
- Não publique payloads destrutivos em issues públicas.
- Não explore falhas em ambientes de terceiros sem autorização.
- Ao contribuir com código, valide entradas, permissões, uploads e saídas HTML para reduzir risco de XSS, CSRF, IDOR e vazamento de dados.

## 📦 Escopo

Esta política se aplica ao código do repositório Getfy, incluindo backend Laravel, frontend Vue, instaladores, scripts de deploy e documentação operacional.

## 🤝 Resposta

Os mantenedores analisarão o relato e poderão solicitar informações adicionais. Correções de segurança devem priorizar baixo risco, compatibilidade com instalações existentes e clareza no histórico de commits.
